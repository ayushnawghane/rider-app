India OTP Setup Guide (MSG91 + DLT)

Prepared for platform owner / operations team

We need DLT approvals before OTP SMS can be delivered reliably in India.

1) Register business on a DLT operator portal

Register as Principal Entity (PE) on one approved DLT portal (Jio, Airtel, Vi, or BSNL). Submit company documents (PAN, GST, legal name, address, authorized contact).

2) Create and approve Sender ID (Header)

Create the sender/header you want messages to come from (example: BLINKC) and wait for approval.

3) Create and approve OTP content template on DLT

Create the OTP message template on the DLT portal.

  • Include your brand name.
  • Use DLT variable placeholders as per portal rules.
  • Content must match the final SMS text exactly.

4) Collect approved IDs from DLT

  • PE ID / Entity ID
  • Header (Sender ID)
  • DLT Template ID for the OTP template

5) Map DLT details in MSG91

In MSG91 dashboard:

  • Map PE ID to sender/header.
  • Create or update OTP template with placeholder ##OTP##.
  • Map the approved DLT Template ID into that MSG91 OTP template.

6) Share required values with developer

  • MSG91_AUTH_KEY
  • MSG91_TEMPLATE_ID (MSG91 OTP template ID, not raw DLT ID)
  • Confirmation that DLT mapping is completed (PE ID, Header, DLT Template ID)

7) Test and verify

  1. Send OTP to a real number.
  2. Check MSG91 OTP logs and status.
  3. If status stays Pending or fails, raise MSG91 support ticket with Request ID.

Why this is required

MSG91 is the SMS gateway, but India telecom delivery requires DLT compliance (entity + sender + approved template).
